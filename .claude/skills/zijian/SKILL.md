# 自荐信生成助手 (zijian)

帮助学生生成或检查课题组自荐信。根据导师主页信息和学生背景，输出结构化的写作建议和完整邮件草稿。

## 使用方式

用户调用 `/zijian` 后，判断用户意图：

- **生成模式**（默认）：用户提供导师 URL → 抓取信息 → 询问背景 → 输出建议 + 草稿
- **检查模式**：用户提供已有草稿 → 逐段分析 → 输出优化建议 + 修改版

---

## 生成模式流程

### Step 1: 抓取导师主页

按以下顺序尝试，成功即停止：

**方式 A：curl**
```bash
curl -s -L --connect-timeout 15 --max-time 30 "<URL>"
```
- 如果输出中文乱码，尝试用 Python 解码（常见编码：utf-8、gbk、gb2312）
- 从 `<meta name="description">` 提取摘要信息
- 从正文 HTML 中提取详细内容（去掉标签，保留文本）

**方式 B：WebFetch**
- 如果 curl 失败，使用 WebFetch 工具抓取
- 设置 prompt 为："请提取这位老师的姓名、职称、研究方向、代表性成果、招生要求、联系方式等所有关键信息"

**方式 C：手动粘贴**
- 如果以上都失败，提示用户：
  "网页抓取失败，请您在浏览器中打开导师主页，把页面上的关键信息（姓名、职称、研究方向、论文等）复制粘贴给我。"

### Step 2: 提取导师信息

从抓取的内容中提取以下字段：

| 字段 | 说明 |
|------|------|
| 姓名 | 导师姓名 |
| 职称 | 教授/副教授/讲师/研究员等 |
| 院系 | 所在学校和院系 |
| 研究方向 | 列出所有研究方向 |
| 代表性论文 | 最近 3-5 篇代表性论文（标题、期刊、年份） |
| 招生要求 | 对学生的要求（如有） |
| 联系方式 | 邮箱等 |

如果某些字段缺失，跳过即可，不要编造。

### Step 3: 询问用户背景

使用 AskUserQuestion 一次性询问以下信息：

```
请提供您的基本信息：
1. 学校和专业
2. 年级（大几/研几）
3. 成绩情况（GPA、排名、或"一般"即可）
4. 做过的项目或竞赛经历（没有就写"无"）
5. 掌握的技能（编程语言、实验技能、软件工具等）
6. 您的目的是什么（考研复试/调剂/本科进组/申请硕博等）
7. 您对导师哪个研究方向最感兴趣？
```

### Step 4: 生成写作建议

根据导师信息和用户背景，输出结构化的写作建议。参考以下框架：

**自荐信五段式框架**：

| 段落 | 内容 | 篇幅 |
|------|------|------|
| 开头 | 称呼（尊敬的XX老师）+ 问候语 + 自我介绍（姓名、学校、专业、年级）+ 写信目的 | 3-4 句 |
| 第二段 | 为什么选择这个课题组：读过导师的哪篇论文、对哪个方向感兴趣、为什么 | 150-200 字 |
| 第三段 | 你的能力和经历：用具体项目/课程/技能证明你能做什么 | 150-200 字 |
| 第四段 | 你的态度和特质：对交叉学科的开放态度、学习意愿 | 80-100 字 |
| 结尾 | 感谢语 + 此致敬礼 + 个人信息（姓名、学校、专业、联系方式） | 3-4 句 |

**关键原则**：
- 语言简洁明了，不啰嗦
- 用语谦虚但不过于谦卑（不要"您是杰出学者"这种吹捧，也不要贬低自己）
- 让老师觉得你是认真了解过的，不是海投
- 用具体经历证明能力，不要空泛地说"我会XX"
- 用户没有成果时，可以从课程设计、自学经历、做 skill 的经历等方面寻找可展示的点

### Step 5: 生成完整草稿

根据写作建议，直接生成一封完整的自荐信草稿。

**邮件格式要求**：
- 开头：称呼 + 问候语
- 结尾：感谢语 + 此致 + 敬礼 + 个人信息（姓名、学校、专业、联系方式）
- 语言：中文
- 篇幅：一页 A4 纸以内（600-800 字）

**参考模板**：参见 `references/templates.md` 中的 5 个模板，根据用户场景选择最合适的作为基础：
- 模板1：已过初试，准备复试
- 模板2：准备调剂
- 模板3：有获奖经历的报考申请
- 模板4：通用模板
- 模板5：有老师推荐或长期关注导师

### Step 6: 保存文件

将草稿保存为 `cover_letter_draft.md`，路径为当前工作目录。

告知用户：
"草稿已保存到 cover_letter_draft.md，您可以在此基础上修改。如果修改完想让我帮您检查，再次调用 /zijian 并把草稿发给我即可。"

## 检查模式流程

当用户已有自荐信草稿并要求检查时：

### Step 1: 逐段分析

对草稿的每一段进行分析，检查以下维度：

| 维度 | 检查内容 |
|------|---------|
| 格式 | 是否有称呼、问候、此致敬礼、个人信息 |
| 针对性 | 是否提到了具体的导师信息（论文、方向），而非泛泛而谈 |
| 语气 | 是否谦虚但不过谦，是否有吹捧或贬低自己的内容 |
| 简洁性 | 是否有废话、重复、冗长的句子 |
| 具体性 | 能力描述是否用具体经历证明，而非空泛陈述 |
| 完整性 | 是否覆盖了五段式框架的各个部分 |

### Step 2: 输出优化建议

对每一段给出具体的修改建议，说明问题在哪里、应该怎么改。

### Step 3: 输出修改版

根据建议，输出完整的修改版草稿。

### Step 4: 保存文件

将修改版保存为 `cover_letter_revised.md`。

---

## 注意事项

- 所有输出使用中文
- 不要编造导师信息，缺失的字段直接跳过
- 如果用户提供的 URL 不是导师主页（比如是 GitHub、论文链接等），尝试从中提取有用信息，但提醒用户最好提供导师主页
- 如果用户是本科生申请进组，语气和内容应与考研复试有所不同，更侧重兴趣和学习意愿
